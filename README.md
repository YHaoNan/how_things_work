## 项目结构

本项目的主要模块：
1. 一个基于`motion-canvas`的视频框架，它封装了大部分内容，且易于被Agent驱动。如果没有它将：
   1. **视觉不统一**：生成的动画项目风格难以受控，不利于营造记忆点
   2. **人力成本高**：没有一个稳定、简单的工作流导致Agent的自由度太高，随之而来的，可能出现的问题也多
   3. **项目难维护**：生成的动画项目代码风格、项目结构都不固定，导致难以维护
2. 一个MCP Server，用于给Agent接入能力，如
   1. 接入视觉模型对生成的视频进行检查验证
   2. 从互联网上检索并下载图片
   3. 调用视频生成模型生成视频片段并引用
   4. ...
3. 一套精心设计的提示词，用于驱动Agent



对于Agent本身，我们依赖Trae或Cursor等提供的Agent，因为：
1. 在经济上更实惠
2. 开箱即用的code-base探索，甚至可以直接探索当前的node_modules中的依赖库，生成的代码质量更高




```text
=========== 核心代码 ===============
src                   视频框架主代码，被设计成易于被Agent驱动，易于生成同样主题、样式的视频的框架
mcp_server            mcp服务器，供主Agent调用以扩展其功能，目前仅提供视觉校验功能
prompt.md             驱动agent的提示词
测试规划.md           agent测试相关技能（后续迁移到skill下）


============ 用户资产 ==============
projects              用户视频项目文件夹，每一个项目都是一个子文件夹


============ 非核心  ==============
scripts               和主功能无关的脚本，如快速生成一个空项目，快速在项目间切换。一般情况下不用关心
example_project       示例视频项目
~~agent~~             已废弃，目前依赖Trae的Agent
```
