# Generate Rules & Limit
1. 读取根目录的`tsconfig.json`，获取当前的工作目录`@ws`
2. 知识分层
   1. 关于框架、类库的通用知识，总结到根目录的`.knowledge/`中
   2. 关于当前项目的知识，总结到项目目录的`.knowledge/`中
3. 文件相关限制
   1. 长度限制：知识的粒度应该足够细，每个文件描述一个专业化的知识，多种知识不应放在一个文件中。单文件不超过2000个字符。
   2. 文件名称限制：使用中文名来描述知识的核心内容，不超过50个字符
   3. 总结先行：在每一个知识文件前首先给出总结性的简短描述，如何时使用。限制在前10行。
4. 谨慎总结：只有必要的，通用性的知识应该被总结，不要过于频繁的总结，因为质量低下或通用性低的知识不仅没有用，而且会扰乱后续的人

# Knowledge Load

1. 获取根目录`.knowledge/`下的所有文件和项目目录下的`.knowledge/`中的所有文件
2. 根据文件名判断是否有你需要的文件
3. 读取所有你需要的文件的前10行，对它们再次进行过滤，判断是否包含你需要的知识
4. 最终读取全部你需要的知识文件
