# 流程简介
1. 每当你生成完一段动画时，你需要调用工具来检查问题，除非用户要求你不要测试这段动画
   1. 运行命令，通过render命令渲染动画，查看是否有运行时报错
   2. 动画是否挂起（编译过程迟迟无法完成）
   3. 现在可以运行服务，暴露9000端口
   4. 顺序检查每一个测试点，调用`check`工具，检查视觉上是否有问题，如果返回有问题，你应该修复后再次调用check，直到不再返回问题，再继续检查下一个测试点。
2. 每次修复完成后，你应该再次进入整个测试流程，直到测试通过

# 视觉测试规则
1. 你应该在需要视觉检测的位置添加测试点，只需要`testPoint('测试点名称', {check: '你的预期'})`即可。别忘了导入`import {testPoint} from '@src/testing/visualTestPoint';`
   ```ts
   testPoint('agent-end', {
      check: '我希望左侧是用户的对话气泡，右侧是AI的多个思考/行动/观察链，下部是工具framework'
   })
   ```
2. 测试点仅用于检测布局问题，因为如果没有视觉，你经常会生成某些排版异常的情况。所以请在动画归于静态后添加测试点，比如在等待某个event时。
3. 生成单个动画时，请在`program.tsx`中仅插入`director.playLayer(待测试Layer)`来快速测试动画，此时你要删除所有其它代码
